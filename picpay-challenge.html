Desafio Pic Pay

Linguagem: Java
Framework: Spring Boot
Tecnologias: JPA, H2, Lombok, Devtools, Openfeign, Swagger e Validation

Este projeto é uma API RESTful desenvolvida em Java e Spring Boot que simula uma conta bancária, semelhante ao PicPay. A API foi projetada para permitir a manipulação de dados da conta do cliente e realizar transações.

Funcionalidades Principais:

Manipulação de Dados do Cliente: A API fornece endpoints para criar, atualizar, recuperar e excluir informações da conta do cliente.

Transações: A API permite que os usuários realizem transações entre contas. Existem restrições específicas para essas transações. Por exemplo, um lojista só pode receber transferências de usuários normais e não pode transferir para outros lojistas. No entanto, um usuário normal pode transferir tanto para lojistas quanto para outros usuários normais.

Integração com APIs Externas:

A API se integra com serviços externos para validar transações e enviar e-mails. Utiliza o OpenFeign, um cliente declarativo da web do Spring, para facilitar a comunicação HTTP com esses serviços externos.

Documentação da API com Swagger:

O projeto também inclui documentação da API usando o Swagger. O Swagger é uma ferramenta que ajuda os desenvolvedores a projetar, construir, documentar e consumir APIs RESTful. Ele gera uma interface de usuário interativa, permitindo que os usuários explorem e testem a API.

Arquitetura do Projeto:

O projeto segue a Clean Architecture, o que significa que o código é organizado de maneira que seja fácil de entender, escalável e fácil de manter. Além disso, essa estrutura facilita a escrita de testes unitários e de integração.
